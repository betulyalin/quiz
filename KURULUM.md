# Quiz Sonuçlarını Kaydetmek İçin Kurulum (Vercel + Supabase)

Arkadaşlarının test sonuçlarını (isim, kaç doğru, hangi soruda ne cevap verdikleri) görebilmen için Supabase kullanıyorsun. Aşağıdaki adımları uygula.

## 1. Supabase hesabı ve proje

1. [supabase.com](https://supabase.com) → Sign up (ücretsiz).
2. **New project** → Proje adı ve şifre ver, region seç, Create.
3. Sol menüden **Table Editor** → **New table**.
4. Tablo adı: `quiz_results` (tam bu isim olmalı).
5. Aşağıdaki sütunları ekle (veya SQL ile tek seferde oluştur):

| Column name | Type | Default / ayar |
|-------------|------|----------------|
| id | int8 | Primary key, auto (Identity) |
| name | text | |
| score | int4 | |
| total | int4 | |
| answers | jsonb | |
| created_at | timestamptz | Default: `now()` |

**Kolay yol – SQL ile tablo:** Sol menü **SQL Editor** → New query, aşağıdakini yapıştır → Run:

```sql
create table if not exists quiz_results (
  id bigint generated by default as identity primary key,
  name text not null,
  score int not null,
  total int not null,
  answers jsonb not null default '[]',
  created_at timestamptz not null default now()
);

-- Sadece service role (Vercel API) yazabilsin; sen Table Editor'dan okuyabilirsin
alter table quiz_results enable row level security;
```

Böylece tablo ve güvenlik ayarı hazır olur.

## 2. Supabase anahtarlarını al

1. Supabase projesinde sol menü **Project Settings** (dişli).
2. **API** bölümünde:
   - **Project URL** → kopyala (`SUPABASE_URL` olacak).
   - **Service role** key (anon değil) → **Reveal** → kopyala (`SUPABASE_SERVICE_ROLE_KEY` olacak).

Service role key’i sadece sunucuda (Vercel’de) kullan, tarayıcıda gösterme.

## 3. Vercel’de environment variables

1. [vercel.com](https://vercel.com) → Projeyi seç (quiz projesi).
2. **Settings** → **Environment Variables**.
3. Şu iki değişkeni ekle:

| Name | Value |
|------|--------|
| `SUPABASE_URL` | Supabase Project URL |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase Service role key |

4. **Save**.
5. Projeyi bir kez **Redeploy** et (Deployments → son deploy → ⋮ → Redeploy).

Bundan sonra her test bittiğinde sonuçlar otomatik olarak `quiz_results` tablosuna yazılacak.

## 4. Sonuçları nerede görürsün?

- Supabase → **Table Editor** → `quiz_results` tablosunu aç.
- Her satır: bir kişinin ismi, `score`, `total`, `answers` (soru soru hangi cevabı verdiği) ve `created_at`.

`answers` sütunu JSON: her eleman `question`, `selectedAnswer`, `correctAnswer`, `isCorrect` bilgisini içerir.

## Notlar

- **Local (npm start):** API sadece Vercel’de çalışır. Local’de test ederken sonuç kaydedilmez; deploy edince kayıt çalışır.
- Service role key’i kimseyle paylaşma ve GitHub’a koyma; sadece Vercel env’te kalsın.
